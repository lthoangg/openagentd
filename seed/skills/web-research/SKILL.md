# Web Research Skill

Use this approach for any task that requires fetching and synthesising
information from the web.

## Search strategy

1. Start with 1-2 precise queries. Avoid broad single-word searches.
2. If first results are thin, reformulate — try synonyms, official terminology,
   or add a site filter (e.g. `site:docs.python.org`).
3. For time-sensitive topics add the current year or "latest" to the query.
4. Fetch primary sources (official docs, papers, press releases) not just
   summaries.

## Source quality

| Signal | Strong | Weak |
|--------|--------|------|
| Authority | Official org, peer-reviewed, primary source | Anonymous blog, forum post |
| Recency | < 3 months | > 1 year for fast-moving topics |
| Corroboration | 2+ independent sources agree | Single source |
| Specificity | Named data, methodology, version numbers | Vague claims |

Never cite a single source for a key claim. When sources conflict, report both
and flag the discrepancy explicitly.

## Output format

For each key finding:
- **Claim**: 1-2 sentence summary of what was found
- **Source**: URL / publication / date
- **Confidence**: High / Medium / Low

End with a short synthesis that directly answers the original question.
